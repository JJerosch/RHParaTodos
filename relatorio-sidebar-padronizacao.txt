3. O QUE FALTA FAZER (inconsistencias nos demais templates)
------------------------------------------------------------

3.1 SEM ICONES (14 templates)
    Todos os templates exceto dashboard.html usam apenas <span class="nav-text">
    sem nenhum <span class="nav-icon"> com imagem. Falta copiar o padrao de
    icones do dashboard.html para:
      - benefits.html
      - employees.html
      - departments.html
      - positions.html
      - recruitment.html
      - training.html
      - performance.html
      - payroll.html
      - vacation.html
      - timesheet.html
      - reports.html
      - settings.html
      - employee-dashboard.html
      - payslips.html

3.2 SEM CONTROLE DE ROLES (14 templates)
    Nenhum template alem do dashboard.html possui atributos data-roles.
    Todos os itens de menu ficam visiveis para qualquer usuario.

3.3 URLs ESTATICAS em vez de Thymeleaf (14 templates)
    Os templates usam href="dashboard.html" (estatico) em vez de
    th:href="@{/dashboard}" (Thymeleaf). Isso causa problemas quando
    o context-path muda ou quando se usa resolucao de rotas do Spring.

3.4 NAO EXISTE FRAGMENT COMPARTILHADO
    Cada template copia/cola o sidebar inteiro. Qualquer alteracao
    no menu precisa ser replicada manualmente em 16 arquivos.
    Nao existe pasta fragments/ nem arquivo sidebar.html reutilizavel.

3.5 meu-perfil.html SEM SIDEBAR
    A pagina de perfil do usuario nao possui sidebar alguma.

6. ARQUIVOS CSS E JS RELACIONADOS
----------------------------------

  CSS:
    /static/styles/global.css      -> Variaveis CSS (cores, espacamento)
    /static/styles/dashboard.css   -> Estilos do sidebar (.sidebar, .nav-item, etc.)
    /static/styles/components.css  -> Componentes reutilizaveis

    /static/scripts/auth.js        -> Auth.applyRoleVisibility() (controle de roles)
    /static/scripts/components.js  -> initSidebar() (hamburger mobile)
    /static/scripts/dashboard.js   -> Logica especifica do dashboard

  CSS do sidebar (dashboard.css):
    .sidebar            -> 260px fixo, fundo escuro, z-index 100
    .nav-item           -> flex, padding, hover
    .nav-item.active    -> fundo claro + borda esquerda 3px branca
    .nav-section        -> uppercase, opacidade reduzida, separador
    .nav-icon           -> container de icone
    .nav-text           -> texto do item

  Responsividade (breakpoint 768px):
    Desktop: sidebar fixa, main-content com margin-left 260px
    Mobile:  sidebar escondida (translateX -100%), botao hamburger visivel
